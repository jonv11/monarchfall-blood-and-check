# Naming Conventions and Style Guide

## Purpose
Define the naming and style rules for MFBC to ensure consistent, readable, and maintainable code.

## Scope
Applies to all source code, tests, and documentation in this repository.

## When to Use
Use this index when creating or reviewing identifiers, naming patterns, or code style in MFBC.

## Critical Rules

- Prioritize clarity over brevity.
- Use English-only identifiers and comments.
- Follow established patterns consistently.
- Align with Microsoft C# coding conventions and this guide's modules.
